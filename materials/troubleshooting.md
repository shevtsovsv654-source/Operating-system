# COMMON MISTAKES / TROUBLESHOOTING
## 1. Confusing Hardware and Software
### Common Mistake
Translating “hardware” and “software” incorrectly or using them interchangeably.
### Incorrect Example
❌ expensive software
(when the original text refers to hardware)
### Correct Example
✅ expensive hardware → дорогостоящее оборудование
### Explanation
In technical English, hardware refers to physical devices, while software refers to programs and applications.

2. Literal Translation of Technical Terms
Common Mistake
Translating IT terminology word-for-word instead of using established technical equivalents.
Incorrect Example
❌ intrusive updates → вторгающиеся обновления
Correct Example
✅ intrusive updates → навязчивые обновления
Explanation
Technical English often uses metaphorical language that should be translated naturally in Russian IT style.
________________________________________
3. Misunderstanding “Bloatware”
Common Mistake
Confusing bloatware with malware.
Incorrect Example
❌ bloatware → вредоносное ПО
Correct Example
✅ bloatware → предустановленное ненужное ПО
Explanation
Bloatware refers to unnecessary pre-installed applications, not malicious software.
________________________________________
4. Incorrect Translation of “Battery Life”
Common Mistake
Translating battery life literally.
Incorrect Example
❌ источник питания
Correct Example
✅ время автономной работы
Explanation
Battery life describes how long a device works without recharging.
________________________________________
5. Confusing “Optimized” and “Designed”
Common Mistake
Replacing “optimized for” with “designed for.”
Incorrect Example
❌ games designed for Windows
Correct Example
✅ games optimized for Windows
Explanation
Optimization refers to performance adjustment and compatibility improvement.

6. Incorrect Translation of “Security Vulnerability”
Common Mistake
Using vague phrases instead of technical cybersecurity terminology.
Incorrect Example
❌ problems with security
Correct Example
✅ security vulnerabilities → уязвимости безопасности
Explanation
A vulnerability is a specific weakness in a system’s protection.

7. Problems with Enterprise Terminology
Common Mistake
Misunderstanding the term “enterprise.”
Incorrect Example
❌ production infrastructure
Correct Example
✅ enterprise infrastructure → корпоративная инфраструктура
Explanation
Enterprise refers to large organizations and business environments, not industrial production.

8. Misinterpreting “Update Loop”
Common Mistake
Literal translation of update loop.
Incorrect Example
❌ обновляющий цикл
Correct Example
✅ зацикливание обновлений
Explanation
An update loop is a situation where the system repeatedly attempts to install updates.
________________________________________
9. Literal Translation of “Seamless Integration”
Common Mistake
Ignoring the technical meaning of seamless.
Incorrect Example
❌ smooth integration
Correct Example
✅ seamless integration → бесшовная интеграция
Explanation
This term describes smooth interaction between systems or devices without interruptions.
________________________________________
10. Confusing “Local Device” and “Internal Memory”
Common Mistake
Replacing “local device” with “internal memory.”
Incorrect Example
❌ внутренняя память устройства
Correct Example
✅ локальное устройство / само устройство
Explanation
A local device refers to the physical user device, not its storage system.

11. Incorrect Interpretation of “Reviewed Applications”
Common Mistake
Confusing reviewed with optimized or debugged.
Incorrect Example
❌ тщательно отлаженные приложения
Correct Example
✅ тщательно проверенные приложения
Explanation
Reviewed applications are applications checked and approved by moderators.

13. Overusing Literal Translation
Common Mistake
Translating English sentence structure directly into Russian.
Incorrect Example

Correct Example
✅ ChromeOS делает акцент на простоте
Explanation
Professional technical translation requires adaptation, not direct word-for-word translation.

15. Problems with Technical Style
Common Mistake
Using overly academic or unnatural constructions in IT texts.
Incorrect Example
❌ система характеризуется высокой степенью эффективности функционирования
Correct Example
✅ система работает эффективно
Explanation
Modern IT English and Russian technical writing usually prefer concise and clear language.
